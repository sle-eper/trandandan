## 🔑 Authentication
Authentication is about verifying **who the user is** before they access the system.

### Responsibilities
- **Register (Sign Up)**: Allow new users to create accounts with email, username, and password.
- **Login (Sign In)**: Validate user credentials to grant access.
- **Password Security**: Hash and salt passwords (e.g., using bcrypt). Never store plain text passwords.
- **Tokens / Sessions**: Manage logged-in users using JWT or cookies/sessions.
- **Two-Factor Authentication (2FA)**: Add an extra layer of security (e.g., email or mobile code).
- **OAuth / 42 API**: Enable login with external providers such as 42 Intra.

👉 Goal: **Ensure secure and reliable user authentication.**
